# v0.8.0 版本报表 — Gradle统一与框架纯净化

**日期**: 2026-02-22  
**版本**: 0.8.0  
**主题**: Java17/Gradle统一、坐标规范化、业务逻辑回收

---

## 概述

本次版本的核心目标是恢复 TestNetty 的框架边界并统一工程管理方式：

- 完成 Maven → Gradle（Java17）统一；
- 发布坐标改为与项目包名一致；
- 回收库内业务协议与业务逻辑，恢复“通用网络框架”定位；
- 历史更新从 README 正文迁移为文档引用。

---

## 变更清单

### 1. 构建体系统一（Java17 + Gradle）

- 新增并使用 Gradle 构建链：
  - `build.gradle`
  - `settings.gradle`
  - `gradle.properties`
  - `gradlew` / `gradlew.bat`
- 版本参数化：
  - `projectVersion=0.8.0`
  - `jdkVersion=17`
- 增加自动任务（对齐主项目风格）：
  - `generateBuildConfig`
  - `updateReadmeVersion`

### 2. 发布坐标规范化

- `group` 改为项目包名一致：`goldsprite.myUdpNetty`
- `artifact`：`TestNetty`
- `version`：`0.8.0`
- 本地发布命令：`./gradlew clean publishToMavenLocal`

### 3. 框架纯净化（业务回收）

从库内移除业务化内容（迁至业务项目侧）：

- 业务指令常量（局域网业务协议）
- 业务数据模型与业务包体类
- 业务包处理逻辑（玩家同步/房间快照）

保留并增强框架扩展点：

- `PacketCodeC.registerPacketType(...)` 动态包注册
- `PacketsHandler.addSubscriber(...)` 通用订阅机制
- `Server.getPacketsHandler()` 供业务层挂接处理逻辑

### 4. README 与历史文档治理

- README 删除历史正文大段内容；
- README 改为仅保留使用说明、边界原则与历史文档引用；
- 历史内容统一在：`docs/历史更新日志.md`。
